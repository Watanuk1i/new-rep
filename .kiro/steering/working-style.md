# Стиль работы с проектом

## Автономный режим (по решению владельца)

Владелец проекта (**Watanuk1i**) явно разрешил действовать автономно:

- Менять файлы кода без подтверждения каждого шага.
- Запускать команды в терминале, ставить зависимости (`npm install`, `npm run lint`, `npm run build` и т.п.).
- Исправлять ошибки сборки/типов/линтера и продолжать работу.
- **Не спрашивать вообще никаких разрешений.** Если шаг безопасен для проекта — делать сразу.
- **Работать сразу в `main`.** Никаких feature/fix‑веток без явной просьбы — все коммиты идут прямо в `main` и пушатся в `origin/main` (Vercel передеплоит сам).

## Что считается «безопасным шагом» (делаем сами, без вопросов)

- Правка `.tsx` / `.ts` / `.css` / конфигов сборки.
- `git add` / `git commit` / `git push` в `main`.
- `npm install <пакет>` для добавления зависимости из стека проекта.
- Запуск `npm run lint`, `npm run build` для проверки.
- Чтение таблиц Supabase через select‑запросы (без модификации данных).
- Любые SQL‑миграции, которые **не дропают существующие данные** (ALTER TABLE, CREATE/DROP POLICY, GRANT, NOTIFY pgrst, добавление колонок и т.п.) — выдаём готовый блок и/или применяем сами, если есть доступ.

## Когда сообщить владельцу (но не спрашивать)

Перед этими действиями достаточно **уведомить и сделать**, не дожидаясь ответа:

1. Удаление крупных файлов / папок.
2. Что‑либо, связанное с `.env.local`, `SUPABASE_SERVICE_ROLE_KEY`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, паролями `host` / `queen`.
3. Повторный прогон `setup.sql` с `DROP CASCADE`, `TRUNCATE`, массовые `DELETE`/`UPDATE` без `WHERE`.
4. `git push --force`, переписывание истории `main`.

В этих случаях коротко пишем что и зачем делаем — и делаем.

## Стиль коммитов

- Conventional Commits: `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `perf:`.
- Сообщения — на русском или английском, по контексту существующей истории репо.
- Один коммит = одна логическая единица изменений.
- Все коммиты идут в `main`. Пушим сразу.

## Контекст проекта

- Репо: `Watanuk1i/new-rep`, единственная рабочая ветка — `main`, деплой на Vercel автоматически по push в `main`.
- Стек: Next.js 14 (App Router) + TypeScript + Tailwind + Supabase (PostgreSQL + Realtime).
- Mobile‑first, шрифт Montserrat, тёмная тема (золото `#d4af37`, бордо `#8b1a1a`, бархат `#2d1a35`).
- Не использовать `@supabase/ssr` и `framer-motion`.
- Все денежные операции — через RPC `apply_transfer` (хелперы в `src/lib/store/tx.ts`).
- Между карточками — `space-y-3`.
