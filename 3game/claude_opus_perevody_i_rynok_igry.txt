Нужно добавить на сайт две связанные экономические функции:

1. Глобальный перевод йен между игроками.
2. Рынок игры для будущей Большой игры “Карточный корабль”.

==================================================
1. ГЛОБАЛЬНЫЙ ПЕРЕВОД ЙЕН
==================================================

Нужно добавить функцию “Перевести йены”.

Назначение:
Игрок может перевести йены другому игроку с обязательным комментарием. Это нужно для сделок, покупок, выплат, компенсаций, взяток и договорённостей внутри игр.

Где добавить:
- в профиле игрока кнопка “Перевести йены”;
- в разделе экономики, если он есть;
- возможно в карточке участника;
- ведущий/админ должен видеть историю переводов.

Поля формы перевода:
- отправитель: текущий игрок;
- получатель: выбор из списка игроков;
- сумма в йенах;
- комментарий / причина перевода;
- привязка к игре, если перевод делается из страницы игры.

Правила перевода:
- сумма должна быть больше 0;
- нельзя перевести больше, чем есть на балансе;
- комментарий обязателен;
- перевод выполняется сразу после подтверждения;
- деньги списываются у отправителя;
- деньги начисляются получателю;
- операция записывается в историю.


Текст записи в истории:
“[Игрок А] перевёл [Игроку Б] ¥[сумма]. Причина: [комментарий].”

Примеры комментариев:
- “Покупка карты Камень в Карточном корабле”
- “Оплата сделки перед выстрелом”
- “Компенсация за место”
- “Возврат долга”
- “Взятка за голос”

Интерфейс:
Кнопки:
- Перевести йены
- Подтвердить перевод
- Отмена

Ошибки:
- “Недостаточно средств”
- “Сумма должна быть больше нуля”
- “Укажите получателя”
- “Комментарий обязателен”

Важно:
Перевод йен не обязан автоматически означать исполнение обещания второй стороны. Если игрок просто перевёл деньги по договорённости, а второй игрок не выполнил обещание, это уже внутриигровой риск, если сделка не была оформлена через рынок.

==================================================
2. РЫНОК ИГРЫ
==================================================

Нужно добавить простую систему “Рынок игры”.

Назначение:
Некоторые Большие игры требуют торговли игровыми предметами. Например, в будущей игре “Карточный корабль” игроки должны покупать и продавать карты, звёзды и другие игровые ресурсы. Рынок нужен, чтобы сайт мог автоматически обменивать предмет на деньги без споров.

На первом этапе рынок нужен только как универсальная механика для одной игры, но лучше сделать его достаточно гибким.

Где добавить:
На странице конкретной игры отдельную вкладку:
- Обзор
- Раунды / дуэли
- Рынок
- История

==================================================
3. РЫНОК ДЛЯ “КАРТОЧНОГО КОРАБЛЯ”
==================================================

Контекст будущей игры:
“Карточный корабль” — Большая игра по мотиву ограниченных камень-ножницы-бумага.

У каждого игрока есть:
- карты Камень;
- карты Ножницы;
- карты Бумага;
- звёзды;
- йены.

Игроки должны иметь возможность торговать картами и звёздами за йены.

Что можно продавать в рынке этой игры:
- карту Камень;
- карту Ножницы;
- карту Бумага;
- звезду.

Пока НЕ добавлять продажу обещаний, услуг, статусов, слухов или прав вызова.

==================================================
4. СОЗДАНИЕ ЛОТА
==================================================

Игрок может создать лот.

Поля лота:
- продавец;
- тип предмета: card / star;
- если card: тип карты rock / scissors / paper;
- количество;
- цена в йенах;
- связанная игра;
- статус лота: active / sold / cancelled;
- время создания.


Правила создания лота:
- продавец должен владеть предметом;
- нельзя выставить больше карт/звёзд, чем есть у игрока;
- цена должна быть больше 0;
- количество должно быть больше 0;
- предметы, выставленные на продажу, лучше временно блокировать, чтобы игрок не мог использовать или продать их дважды.

Кнопки:
- Создать лот
- Выставить карту
- Выставить звезду
- Отменить лот

==================================================
5. ПОКУПКА ЛОТА
==================================================

Покупатель нажимает “Купить”.

Правила покупки:
- покупатель не может купить свой собственный лот;
- у покупателя должно хватать йен;
- лот должен быть активен;
- после покупки деньги списываются у покупателя;
- деньги начисляются продавцу;
- предмет передаётся покупателю;
- лот получает статус sold;
- операция записывается в историю игры и общую историю экономики.

Текст записи в истории:
“[Игрок А] купил у [Игрока Б] [предмет] за ¥[сумма] в игре ‘Карточный корабль’.”

Примеры:
“Макото купил у Бьякуи карту Бумага за ¥70 000 в игре ‘Карточный корабль’.”
“Кёко купила у Джунко 1 звезду за ¥200 000 в игре ‘Карточный корабль’.”

==================================================
6. ОТМЕНА ЛОТА
==================================================

Продавец может отменить активный лот, если он ещё не куплен.

После отмены:
- лот получает статус cancelled;
- заблокированный предмет возвращается продавцу как доступный;
- история фиксирует отмену.

Текст истории:
“[Игрок] отменил лот: [предмет] за ¥[сумма].”

==================================================
7. ОТЛИЧИЕ ПЕРЕВОДА ОТ РЫНКА
==================================================

Важно сохранить два разных действия:

1. Перевод йен — простой перевод денег без гарантии встречного предмета.
2. Рынок — автоматический обмен: деньги на предмет.

Пример:
Если игрок просто перевёл ¥50 000 с комментарием “за карту”, сайт не обязан автоматически передать карту.
Если игрок покупает карту через рынок, сайт автоматически меняет деньги на карту.

Это важно для блефа и сделок:
- простой перевод может быть рискованной договорённостью;
- рынок является безопасной технической сделкой.

==================================================
8. ИСТОРИЯ
==================================================

Все операции должны попадать в историю.

Типы записей:
- money_transfer
- market_listing_created
- market_listing_cancelled
- market_purchase

==================================================
9. ПРАВА ВЕДУЩЕГО / АДМИНА
==================================================

Ведущий должен иметь возможность:
- видеть все переводы;
- видеть все лоты рынка;
- отменить спорный лот;
- вручную откатить операцию только в аварийной ситуации;
- вручную передать карту или звезду, если сайт сломался;
- видеть историю экономики.

Кнопки для ведущего:
- Отменить лот
- Просмотреть историю операций
- Ручная корректировка баланса
- Ручная передача предмета

Важно:
Ручные действия ведущего тоже должны записываться в историю.
