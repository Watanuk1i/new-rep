# Supabase migrations

Два файла:

1. **setup.sql** — первоначальная установка БД с нуля (схема + RLS + RPC + seed).
   Запускается **один раз** при старте проекта.

2. **migration_full_v4.sql** — финальная итеративная миграция.
   Объединяет все улучшения экономики (Фонд Тогами 20M, Кредитный резерв Кируми 6M),
   состав 14 активных игроков, договоры Кируми, кредиты v2, SOS, игры на долг,
   разблокировку Больших игр, новые системные пароли host/queen.
   **Идемпотентна** — можно запускать повторно без потери данных.

3. **fix_minority_vote_alive.sql** — точечный fix RPC для «Правила меньшинства».

## Порядок применения

При первом разворачивании:
```sql
-- 1. setup.sql (создаст схему)
-- 2. migration_full_v4.sql (актуализирует под спецификацию)
-- 3. fix_minority_vote_alive.sql (по необходимости)
```

При обновлении уже работающего проекта:
```sql
-- migration_full_v4.sql (повторно безопасно)
```
